# openclaw-audit -- Thorough Demo & Testing Guide

All scan commands write output to a single `scan_output/` folder in the
project root. This folder is listed in `.gitignore` so it never pollutes
the repo. You can delete it at any time and regenerate it by re-running a scan.

## Part 8 -- Test Correlation Logic (3 min)

**Goal:** Show the tool doesn't just do simple regex -- it thinks across files.

First, regenerate full output for inspection:

```bash
python -m openclaw_audit scan tests/fixtures
```

### Step 8.1 -- Spot correlated findings in the JSON

```bash
python -c "import json, pathlib; findings=json.loads(pathlib.Path('scan_output/findings.json').read_text())['findings']; corr=[f for f in findings if f['id'].startswith('CORR-')]; print(f'Correlated findings: {len(corr)}'); [print(f'  - {c[\"id\"]}: {c[\"title\"]}') for c in corr]"
```

**Expected:** You should see 2-3 correlated findings:
- `CORR-ADMIN-*` -- "Correlated: publicly exposed admin / API with credential risk"
- `CORR-PLUGIN-*` -- "Correlated: plugin loading with execution primitives"
- `CORR-SHARED-*` -- "Correlated: shared privileged agent with weak isolation"

**What to explain to the team:**
- These are *not* direct regex matches
- The correlator noticed that multiple independent signals across different files
  combine to form a higher-level risk
- Example: `bind_address=0.0.0.0` (from JSON) + `admin_panel_port` (from JSON) +
  `api_key=sk-...` (from .env) = "exposed admin surface with credential risk"

## Part 9 -- Test Risk Scoring (2 min)

### Step 9.1 -- Show the scoring math

The risk score uses this formula:
- Base scores: info=1, low=3, medium=6, high=10, critical=15
- Confidence multipliers: high=1.0, medium=0.75, low=0.5
- Total is clamped to 0-100
- Bands: 0-19=Low, 20-39=Moderate, 40-69=High, 70-100=Critical

### Step 9.2 -- Verify with the unit tests

```bash
python -m pytest tests/test_risk.py -v
```

**What to point out:**
- `test_empty_findings` -- score is 0, band is "Low"
- `test_single_critical` -- 15 * 1.0 = 15 (still "Low" because < 20)
- `test_multiple_findings_accumulate` -- 5 high findings = 50 ("High")
- `test_confidence_reduces_score` -- same severity but low confidence scores less
- `test_clamped_to_100` -- 20 critical findings clamp to exactly 100 ("Critical")
- `test_category_breakdown` -- per-category subtotals are correct
